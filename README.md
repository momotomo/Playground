# Rust Paint Foundation

`egui + eframe` だけで構成した、Rust オンリーのお絵かきツールです。  
同じコードベースから native 実行と WebAssembly 実行を扱い、GitHub Pages へ静的配信できます。  
MVP 完成版として、再編集用 `JSON保存`、共有用 `PNG書き出し` / `透過PNG`、図形や線の再利用向け `SVG書き出し`、`ズーム / パン`、`図形再編集`、`ドラッグ矩形選択`、`複数選択の一括リサイズ / 回転`、`グループ化 / グループ解除`、`整列 / 等間隔配置`、`重なり順操作`、`レイヤー`、`グリッド / ガイド / スナップ`、`スマートガイド`、`ルーラー UI` に加えて、`スポイト`、`バケツ塗り`、`線色 / 塗り色`、`不透明度`、`図形の塗り`、`最近使った色`、`ペン / えんぴつ / マーカー` をまとめています。

## Web デモ

- 公開 URL: [https://momotomo.github.io/Playground/](https://momotomo.github.io/Playground/)

## アプリ概要

- Rust だけで UI とアプリケーション本体を構築
- `egui` による即時モード UI
- `eframe` による native / web 共通アプリ基盤
- GitHub Pages でそのまま試せる Web 版

## 主な機能

- フリーハンド描画、消しゴム、`元に戻す`、`やり直す`、`クリア`
- 編集用 JSON 形式での `JSON保存` / `JSONを開く`
- 共有用 `PNG書き出し` / `透過PNG` / `SVG書き出し`
- 左 / 右サイドバーの縦スクロール対応
- キャンバスのズーム、パン、表示リセット
- `スポイト` によるキャンバス色の取得
- `バケツ塗り` による閉じた領域の塗り
- `線色` / `塗り色` の分離と `不透明度` 調整
- `最近使った色` と簡易パレットで色を再利用
- 単一選択と複数選択
- キーボード不要の `複数選択モード` と `手のひら` ツール
- タブレット向けの `ピンチズーム` と `2本指パン`
- タブレット向けの `長押し選択` と、ベストエフォートの `ペン / 指` 役割分離
- 単一選択での移動、リサイズ、回転
- 複数選択での一括移動、グループリサイズ、グループ回転、グループ化 / グループ解除、整列、等間隔配置、重なり順変更
- 矩形、楕円、直線ツール
- フリーハンド用の `ペン` / `えんぴつ` / `マーカー`
- 矩形 / 楕円の塗りあり / なし
- レイヤーの追加 / 削除 / 名前変更 / 表示切替 / ロック / 並び替え
- レイヤーの複製と、作業レイヤーの見やすいカード表示
- グリッド表示、ガイド表示、グリッド間隔調整、グリッド / ガイドスナップ
- ブラシ / 図形用の `描く太さ` と `消しゴム太さ` の個別調整
- スマートガイド表示とルーラー UI
- 初回だけ出るスキップ可能なミニチュートリアル
- 主要操作のツールチップと `?` 補助導線
- native / web の保存導線差分吸収
- OFL ライセンスの `Zen Kaku Gothic New` を同梱し、native / web の両方で日本語 UI を表示

## 技術選定理由

- `egui`
  - Rust だけで UI を完結でき、ツールパネルや状態表示を素早く試作しやすい
- `eframe`
  - `egui` 公式フレームワークで、native と wasm を同じアプリ本体で扱いやすい
- `serde`
  - 作品データを堅実にシリアライズするため
- `serde_json`
  - 再編集用の独自保存形式を読みやすい JSON envelope で持たせるため
- `rfd`
  - native のファイルダイアログと web のダウンロード / ファイル選択を Rust だけで扱うため
- `tiny-skia`
  - 作品データから表示倍率に依存しない PNG をラスタライズするため
- `Trunk`
  - wasm ビルドとローカル確認を簡潔に行うため
- `Zen Kaku Gothic New`
  - OFL で同梱しやすく、egui の fallback font として Web でも日本語グリフを安定表示するため

## Rust バージョン要件

- `rust-toolchain.toml` で `1.94.0` に固定しています
- `Cargo.toml` 上の最小要件は `rust-version = 1.88`
- `egui` / `eframe` は breaking changes が入りやすいため、現時点では `0.33.3` に固定しています

## セットアップ手順

### 1. Rust を入れる

```bash
rustup toolchain install 1.94.0
rustup default 1.94.0
rustup component add rustfmt clippy
rustup target add wasm32-unknown-unknown
```

### 2. Trunk を入れる

```bash
cargo install --locked trunk --version 0.21.14
```

### 3. 依存解決

```bash
cargo fetch
```

## native 起動手順

```bash
cargo run
```

## native 版の使い方

- 起動したら `ペン`、`えんぴつ`、`マーカー` か図形ツールを選んでドラッグすると描けます
- 左パネルは縦スクロールできるので、小さい画面でも下の設定まで操作できます
- `線色` は描画ツールと図形、`塗り色` は四角形・楕円・バケツ塗りに使います
- `描く太さ` は `ペン / えんぴつ / マーカー / 図形`、`消しゴム太さ` は消しゴムに使います
- `最近使った色` と簡易パレットから、線色や塗り色をすぐ呼び戻せます
- `不透明度` を下げると、線や塗りを半透明にできます
- `四角形`、`楕円`、`直線` を 1 つ選ぶと、左パネルからその図形の線色 / 塗り / 太さ / 不透明度を直接調整できます
- 編集を続けるなら `JSON保存`、見たまま共有するなら `PNG書き出し`、透明素材なら `透過PNG`、図形や線を再利用したいなら `SVG書き出し` を使います
- `JSONを開く` は OS のファイルダイアログから再編集用データを開きます

## web 起動手順

### ローカル開発サーバー

```bash
trunk serve
```

- 既定の URL は `http://127.0.0.1:8080`
- ブラウザで上記 URL を開くと WebAssembly 版が動作します

### 配布用ビルド

```bash
trunk build --release
```

- 出力は `dist/`
- GitHub Pages へ載せる静的成果物になります

## Web 版の使い方

- 公開 URL か `trunk serve` のローカル URL をブラウザで開きます
- 初回は短い `ミニチュートリアル` が出ます。スキップしても、上部の `ヘルプ` から再表示できます
- 主要ボタンはホバーで短い説明を表示し、タブレットでは見出し横の `?` から補助説明を出せます
- `JSON保存` はブラウザダウンロード、`JSONを開く` はファイル選択、`PNG書き出し` / `透過PNG` / `SVG書き出し` はダウンロードです
- GitHub Pages 上でも描画、選択、変形、レイヤー、グリッド / ガイドはそのまま使えます
- タブレットでは `複数選択モード`、`手のひら` ツール、`ピンチズーム`、`2本指パン`、`長押し選択` を使うと、キーボードなしでもかなり自然に操作できます

## 最初の使い方

1. `ペン`、`えんぴつ`、`マーカー`、`四角形`、`楕円`、`直線` のどれかを選び、キャンバスをドラッグして 1 つ描きます。
2. 左パネルで `線色`、必要なら `塗りを使う`、`塗り色`、`不透明度` を調整します。`最近使った色` や簡易パレットから戻すこともできます。
3. `スポイト` を使うと、キャンバス上の見えている色を拾って `線色` または `塗り色` に反映できます。
4. `バケツ塗り` を使うと、クリック / タップした地点から閉じた領域を塗れます。塗り結果は現在のレイヤーに追加されます。
5. `選択` に切り替えてクリックすると、移動や再編集ができます。
6. `Shift + Click` または空き領域ドラッグで複数選択できます。タブレットでは `複数選択モード` をオンにすると、タップで追加 / 解除できます。選び終わったら `選択を保ったまま移動へ` で通常の移動へ戻れます。
7. `手のひら` ツール、`Space + Drag`、または中ボタンドラッグでパンし、`Ctrl/Cmd + Wheel` か `+ / -` でズームします。タブレットでは `2本指ドラッグ` でパン、`ピンチ` でズームできます。
8. タブレットでは既定で、指はビュー操作と `長押し選択` 寄り、ペンやマウスは描画寄りです。指で描きたいときは左パネルの `指でも描く` をオンにします。
9. 編集を続けるなら `JSON保存`、見たまま共有するなら `PNG書き出し`、透過素材なら `透過PNG`、図形や線を再利用したいなら `SVG書き出し` を使います。
10. 操作に迷ったら上部の `ヘルプ` を開くと、最小ヘルプ、ショートカット、`チュートリアルをもう一度見る` を確認できます。

## 操作方法

### ツール

- `選択`
  - 要素をクリックして選択します
- `手のひら`
  - キーボードなしでキャンバスをドラッグして移動します
- `ペン`
  - くっきりした線を描きます
- `えんぴつ`
  - 少し軽く、やや薄く、少しラフな線を描きます
- `マーカー`
  - 半透明で重ねやすく、中心が少し濃く出る線を描きます
- `スポイト`
  - キャンバス上で見えている色を拾って、`線色` または `塗り色` に反映します
- `バケツ塗り`
  - クリック / タップした地点から、見えている作品結果を基準に閉じた領域を塗ります
- `消しゴム`
  - キャンバス背景色で消すフリーハンドツールです
- `四角形`
  - ドラッグした範囲に、線と必要なら塗りを付けた四角形を描きます
- `楕円`
  - ドラッグした範囲に、線と必要なら塗りを付けた楕円を描きます
- `直線`
  - ドラッグ開始点から終了点まで直線を描きます

### 左パネル

- 左パネルは縦スクロールできます
- 小さい画面やタブレットでも、下の `配置補助` や `保存と書き出し` までスクロールして操作できます
- `描画ツール設定` では `線色` / `塗り色` / `不透明度` と `描く太さ` / `消しゴム太さ` を調整できます
- `バケツ塗り` は `塗り色` とその不透明度を使います
- 単一の `四角形` / `楕円` / `直線` を選ぶと、同じ場所が「選択中の図形の設定」として働きます
- `最近使った色` と `よく使う色` から、現在の反映先へ色をすぐ入れられます
- `塗りを使う` をオンにすると、四角形と楕円に塗り色を使えます
- `線だけ` / `線と塗り` で、図形の見え方をすばやく切り替えられます
- `色の反映先` で、スポイト・最近使った色・簡易パレットの色を `線色` と `塗り色` のどちらへ入れるか選べます

### 選択 / 再編集

- `選択` ツールでストロークと図形のどちらも選択できます
- 最小レイヤー実装では、選択と描画の対象は「現在のレイヤー」のみです
- 非表示のレイヤーは描画、選択、PNG 出力に含まれません
- ロック中のレイヤーは表示されますが、選択や編集の対象になりません
- 選択中要素は `ここへ移動` / `ここへ複製` で別レイヤーへ送れます
- 移動先 / 複製先には、表示中かつロックされていないレイヤーだけを使えます
- レイヤー間移動 / 複製の完了後は、移動先レイヤーが現在のレイヤーになり、新しい要素群の選択を維持します
- `Shift + Click` で選択に追加 / 解除できます
- `複数選択モード` をオンにすると、キーボードなしでタップ / クリックによる追加選択と解除ができます
- `複数選択モード` で選び終わったら `選択を保ったまま移動へ` を押すと、選択を保ったまま通常のドラッグ移動へ戻れます
- タブレットでは描画系ツール中でも、指の `長押し` から選択と移動へ入りやすくしています
- 空き領域をドラッグすると矩形選択できます
- `Shift` または `複数選択モード` を使った矩形選択では、既存選択へ追加できます
- 単一選択中の図形にはハイライト付きアウトライン、角ハンドル、回転ハンドルを表示します
- 複数選択中は各要素のハイライト、グループバウンディングボックス、グループ用リサイズ / 回転ハンドルを表示します
- 単一選択の図形は次の再編集に対応します
  - 移動
  - リサイズ
  - 回転
- 単一選択の `四角形` / `楕円` / `直線` は、左パネルから線色 / 塗り / 太さ / 不透明度も直接変更できます
- 複数選択は次の編集に対応します
  - 一括移動
  - 一括リサイズ
  - 一括回転
  - グループ化
  - 横方向 / 縦方向の等間隔配置
  - 左 / 横中央 / 右 / 上 / 縦中央 / 下揃え
  - 前面 / 背面 / 前へ / 後ろへ の重なり順変更
- `グループ化` は複数選択を 1 つの最上位要素にまとめます
- グループ化した要素は 1 つのまとまりとして選択、移動、リサイズ、回転、整列、重なり順変更できます
- `グループ解除` は選択中のグループを 1 階層だけ展開します
- グループ内部の要素順は保持され、描画順と保存順にもそのまま反映されます
- 複数選択のリサイズはグループ bbox を基準に、要素同士の相対位置を保ちながら行います
- 複数選択の回転はグループ中心回りで行います
- ストロークもこのフェーズでは簡易的な一括スケール / 回転対象に含めています
- 角ハンドルをドラッグするとリサイズします
- 回転ハンドルをドラッグすると中心回りに回転します
- ドラッグ中はプレビューし、リリース時に履歴へコミットします
- `Esc` で進行中の編集プレビューをキャンセルできます
- 選択状態そのものは `元に戻す / やり直す` に含めません

### 整列 / 重なり順

- `整列` メニューは複数選択時のみ有効です
- 整列基準は選択要素全体の bounding box です
- 整列では rotation は保ち、位置だけを調整します
- `等間隔` メニューは 3 要素以上の複数選択時のみ有効です
- `横方向に等間隔` / `縦方向に等間隔` は、先頭と末尾の要素を基準に中間要素の間隔を均等化します
- `重なり順` メニューは単一選択でも複数選択でも使えます
- 複数選択の重なり順変更では、選択要素どうしの相対順を保ったまま前後へ移動します

### 編集

- `元に戻す`: 直前の編集を戻します
- `やり直す`: `元に戻す` した編集を戻します
- `クリア`: 作品全体を消去します
- `JSON保存`: 再編集用 JSON を保存します
- `JSONを開く`: JSON から再編集状態を復元します
- `PNG書き出し`: 背景と全要素を含む共有用 PNG を書き出します
- `透過PNG`: 背景を透明にした PNG を書き出します
- `SVG書き出し`: 図形や線を中心に、拡大や再利用に向く SVG を書き出します
- 主要ボタンには短いツールチップがあります。タブレットでは見出し横の `?` から補助説明を出せます

### レイヤー

- 右側の `レイヤー` パネルでレイヤーを管理します
- `レイヤー追加` で新規レイヤーを追加し、そのレイヤーを現在のレイヤーにします
- `複製` で現在のレイヤーをコピーして、そのコピー側を現在のレイヤーにできます
- `レイヤー削除` は現在のレイヤーを削除します
- 最低 1 レイヤーは必ず残ります
- レイヤー名は `レイヤー名を変更` で変更できます
- `表示` / `非表示` で表示切替、`ロック` / `ロック解除` で編集可否を切り替えます
- `上へ` / `下へ` でレイヤー順を並び替えます
- 現在のレイヤーは強調表示され、`作業中` / `非表示` / `ロック中` と要素数をレイヤーカード上に表示します
- 選択があると、他の編集可能レイヤーに `ここへ移動` / `ここへ複製` を表示します
- レイヤー間移動と複製は、移動先レイヤーの末尾へ追加する単純なルールです
- 要素の重なり順は「同一レイヤー内」で維持され、レイヤー順はその外側の描画順として効きます

### グリッド / ガイド / スナップ / ルーラー

- 左パネルの `配置補助` から表示と吸着を切り替えます
- `ルーラーを表示` でキャンバスに追従する上端 / 左端ルーラーを表示します
- `スマートガイド` を有効にすると、要素移動中に他要素の左 / 中央 / 右、上 / 中央 / 下へ揃う候補線を表示し、近ければ吸着します
- `グリッドを表示` はキャンバスの補助グリッド表示です
- `グリッドに吸着` は移動、図形作成、単一 / 複数リサイズ時にグリッドへ吸着します
- `-` / `+` と間隔プリセットでグリッド間隔を調整できます
- `ガイドを表示` は水平 / 垂直ガイドの表示切替です
- `ガイドに吸着` は表示とは独立した吸着設定です
- `横ガイド追加` / `縦ガイド追加` は、選択中なら選択 bbox の中心、未選択ならキャンバス中央にガイドを追加します
- 表示中のガイドはキャンバス上でドラッグして位置を変更できます
- ガイド付近にホバーすると線が少し強調され、ドラッグ可能だと分かるようにしています
- ガイド一覧の `削除` で削除できます
- スナップ対象は bbox の辺 / 中心とドラッグ中ハンドルを基本にしています
- ルーラー、グリッド、ガイド、スマートガイドは補助表示で、PNG には含めません

### ズーム / パン

- `+` / `-` ボタンでズーム
- `表示をリセット` でキャンバス全体が入る表示に戻します
- `Ctrl/Cmd + マウスホイール` でズーム
- `手のひら` ツールでキーボードなしのパンができます
- `Space + Drag` または `中ボタンドラッグ` でパン
- タブレットでは `2本指ドラッグ` でパンし、`ピンチ` でズームできます
- タブレットでは既定で、指はビュー操作 / 長押し選択寄りです。指でも描きたいときは `指でも描く` をオンにします
- 現在のズーム倍率は上部ツールバーと左パネルに表示

### ショートカット一覧

- `Ctrl/Cmd + Z`: 元に戻す
- `Ctrl/Cmd + Shift + Z`: やり直す
- `Ctrl/Cmd + Y`: やり直すの代替
- `Ctrl/Cmd + S`: JSON保存
- `Ctrl/Cmd + O`: JSONを開く
- `Ctrl/Cmd + Shift + E`: PNG書き出し
- `Ctrl/Cmd + G`: グループ化

## 保存形式

### JSON 保存

- 用途は「再編集用」
- 既定ファイル名は `untitled.paint.json`
- 現在の format version は `4` です
- 旧 `version = 1` の stroke-only JSON も読込互換を残しています
- 旧 `version = 2` と `version = 3` の、フラットな stroke / shape / group JSON も読込互換を残しています
- `document.layers[]` にレイヤーを保存します
- 各レイヤーは次の情報を持ちます
  - `id`
  - `name`
  - `visible`
  - `locked`
  - `elements[]`
- グループは `elements[]` を再帰的に保持します
- shape では次の情報を保持します
  - `kind`
  - `color`
  - `fill_color`
  - `width`
  - `start`
  - `end`
  - `rotation_radians`
- 要素の並び順は各レイヤー内の `elements[]` 順として保持され、重なり順としてそのまま復元されます
- レイヤーの並び順も保存され、描画順としてそのまま復元されます
- レイヤー間移動 / 複製の結果も、最終的な各レイヤーの `elements[]` 構成としてそのまま保存されます
- グリッド表示、ガイド表示、グリッド間隔、スマートガイド表示、ルーラー表示、スナップ設定、ガイド位置も同じ `document` 内に保存されます
- 旧 shape JSON に `rotation_radians` がない場合は `0` として読み込みます
- バケツ塗り結果は scanline span ベースの `Fill` 要素として保存され、再編集や PNG 出力へそのまま反映されます

### PNG 出力

- 用途は「共有 / 閲覧用」
- 既定ファイル名は `untitled.png`
- 表示中のズーム倍率や選択枠、ハンドルは含めません
- `PNG書き出し` は作品のキャンバスサイズを基準に、背景色と全要素をラスタライズします
- `透過PNG` は背景 alpha 0 のまま書き出します
- 表示中のレイヤーだけを出力します
- `locked` は表示だけに影響せず、可視なら出力されます
- グリッド、ガイド、スナップ用の補助表示は出力に含めません
- 線色 / 塗り色 / 不透明度もそのまま出力へ反映されます
- バケツ塗り結果も見たままの面としてそのまま出力へ反映されます
- 回転やリサイズ後の図形も、そのまま出力へ反映されます
- 複数選択による整列結果、等間隔配置、グループ変形、重なり順変更、グループ化結果も、そのまま出力へ反映されます
- 透過PNG では、消しゴムは透明に抜く方向で出力されます

### SVG 出力

- 用途は「図形 / 線の再利用や拡大」
- 既定ファイル名は `untitled.svg`
- UI 補助表示、選択枠、ハンドル、グリッド、ガイドは含めません
- `直線`、`四角形`、`楕円` は SVG の線と塗りへ反映します
- freehand は安全な path として簡略化して出力します
- バケツ塗り結果は 1px 高の矩形列として簡略化して出力します
- `ペン / えんぴつ / マーカー` の質感差は SVG では簡略化されます
- `消しゴム` ストロークは SVG では省略します。見たままの共有には `PNG書き出し` / `透過PNG` が向いています
- 表示中のレイヤーだけを出力します
- 線色 / 塗り色 / 不透明度 / 回転 / リサイズ結果は可能な範囲で反映します

## native / web の違い

- native
  - `JSON保存` / `JSONを開く` は OS のファイルダイアログを使います
  - `PNG書き出し` / `透過PNG` は OS ダイアログから `.png` を保存します
  - `SVG書き出し` は OS ダイアログから `.svg` を保存します
- web
  - `JSON保存` はブラウザダウンロードとして JSON を保存します
  - `JSONを開く` はブラウザのファイル選択を使います
  - `PNG書き出し` / `透過PNG` / `SVG書き出し` はブラウザダウンロードとして保存します
  - GitHub Pages 上ではブラウザ制約のため、native のような継続的ファイルハンドル保持はしません
- 上部の `ヘルプ` と空状態の案内は native / web の両方で同じです
- 選択 / 矩形選択 / 複数選択モード / 手のひらツール / グループ化 / グループ解除 / 一括リサイズ / 一括回転 / 整列 / 等間隔配置 / 重なり順変更 / 図形再編集 / レイヤー操作 / グリッド / ガイド / ズーム / パンの基本操作は native / web で同じです

## GitHub Pages デプロイ手順

### 前提

- GitHub Free で使う場合は、リポジトリを `public` にするのが基本です
- private repo では、プラン次第で GitHub Pages が使えないことがあります
- このリポジトリには `.github/workflows/pages.yml` を追加済みです
- 公開後の URL は project pages なら `https://<owner>.github.io/<repo>/` になります

### GitHub 側設定

1. GitHub に push する
2. `Settings` -> `Pages` を開く
3. `Build and deployment` の `Source` を `GitHub Actions` にする
4. `master` へ push すると Pages workflow が走る
5. Actions 完了後、公開 URL を開いて web 版を確認する

## Trunk の色設定エラーを避けるメモ

- 環境変数 `NO_COLOR=1` が入っているシェルでは、`trunk 0.21.14` が `invalid value '1' for '--no-color'` で失敗することがあります
- その場合は次のように上書きしてください

```bash
NO_COLOR=false trunk serve
NO_COLOR=false trunk build --release
```
